Toy Vscode Extension - Vibe Check

发表于2026-06-16|更新于2026-06-17|Other
|阅读量:

最近 Vibe Coding 用量越来越大，古法编程日渐式微。但我有仿古之心，每次 Vibe Coding 完的代码我都自己审过一遍才敢放心用。

但是，我往往只来得及review最关键的部分，且很容易看过就忘，毕竟不是自己写的代码。加上下次 Vibe Coding 可能又把 review 过部分的代码给修改了，导致经常需要 re-review。

git 没法为我区分 AI codes 与 manual codes，review 债越积越大自己写的项目都不好维护。我尝试写文档，但感觉费力不讨好不适合快速迭代，于是写了一个插件，直接做 block 级的跟踪，界定代码中的人类可信边界（

能用就行

其实就是给每个 block 打个 review 过的标，记录行哈希以跟踪改变，避免 re-review 的情况发生。感觉还挺好的，特别是针对较大的 REPO。后续想整个小备忘录，毕竟很多东西不方便写在注释里。
